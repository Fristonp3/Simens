# Project 工程编译与烧录教程（GD32F470VET6 / CMake / OpenOCD / ST-Link）

本文适用于当前工程目录：

`/home/fristonp/Simens/Project`

工程目标芯片为 **GD32F470VET6**（512KB Flash, 256KB SRAM），CMake 目标名为 `gd32f470_industrial_acq`，生成文件为 `gd32f470_industrial_acq.elf/.hex/.bin/.map`。

> 当前仓库里的 `scripts/*.sh` 默认没有执行权限。下面优先给出可直接运行的命令；如需使用 `./scripts/...` 形式，请先执行 `chmod +x scripts/*.sh`。

`/home/fristonp/Simens`

该目录下至少应包含：

- `Firmware/CMSIS/GD/GD32F4xx/` — CMSIS 及启动文件

## 3. 编译固件

### 3.1 一键编译（推荐）

```bash
cd /home/fristonp/Simens/Project
cmake --fresh -S . -B build -G Ninja \
  -DCMAKE_TOOLCHAIN_FILE=cmake/toolchain-arm-none-eabi.cmake \
  -DGD32_SDK_ROOT=/home/fristonp/Simens
cmake --build build
```

编译成功后会在 `build/` 目录下生成：

| 文件 | 说明 |
|------|------|
| `gd32f470_industrial_acq.elf` | ELF 可执行文件（含调试信息） |
| `gd32f470_industrial_acq.hex` | Intel HEX 格式 |
| `gd32f470_industrial_acq.bin` | 纯二进制固件 |
| `gd32f470_industrial_acq.map` | 链接映射文件 |

### 3.2 一键脚本

仓库已提供 `scripts/build_flash.sh`，默认执行 SDK 检查 → CMake 配置 → 编译 → ST-Link 烧录，日志保存到 `logs/` 目录：

直接运行（编译 + 烧录）：

```bash
cd /home/fristonp/Simens/Project
bash scripts/build_flash.sh
```

仅编译不烧录：

```bash
bash scripts/build_flash.sh --build-only
```

仅烧录已有固件：

```bash
bash scripts/build_flash.sh --flash-only
```

> 注意：`scripts/build_flash.sh` 和 `scripts/configure_build.sh` 中的固件名为旧版 `gd32f450_template`，使用脚本前需将其中 `.elf` 文件名替换为 `gd32f470_industrial_acq`，或直接使用上方的 cmake 命令。

## 4. 使用 ST-Link 烧录

完成编译后执行：

```bash
cd /home/fristonp/Simens/Project
cmake --build build --target flash
```

该命令内部等价于：

```bash
openocd -f /home/fristonp/Simens/Project/openocd_stlink.cfg \
  -c "program build/gd32f470_industrial_acq.elf verify reset exit"
```

> `flash` 目标只有在 CMake 配置时已安装 `openocd` 才会生成。如果提示目标不存在，先安装 `openocd` 再重新执行 CMake 配置。

## 5. WSL 下 ST-Link 常见问题

### 5.1 报错 `Error: open failed`

OpenOCD 无法打开 ST-Link USB 设备，优先检查 USB 透传和权限。

先看 WSL 是否识别设备：

```bash
lsusb | grep -i -E "stlink|st-link|stmicro|0483"
```

最小连接测试：

```bash
openocd -f interface/stlink.cfg -f target/stm32f4x.cfg -c "init; shutdown"
```

如果 WSL 看不到设备，在 Windows 管理员 PowerShell 中执行：

```powershell
usbipd list
usbipd bind --busid <BUSID>
usbipd attach --wsl --busid <BUSID>
```

然后回到 WSL 再测试。

如果能看到设备但仍然 `open failed`，可先用 root 验证是否为权限问题：

```bash
sudo openocd -f /home/fristonp/Simens/Project/openocd_stlink.cfg \
  -c "program build/gd32f470_industrial_acq.elf verify reset exit"
```

### 5.2 连接不稳定

把 `openocd_stlink.cfg` 中的 `adapter speed` 从 `4000` 调低到 `1000` 再试。

## 6. 烧录说明

### 固件内存布局

| 区域 | 起始地址 | 大小 | 说明 |
|------|----------|------|------|
| Bootloader | 0x08000000 | 32KB | 串口升级固件 |
| Application | 0x08008000 | 480KB | 主应用程序 |
| 总计 | — | 512KB | 芯片 Flash 容量 |

当前工程将 Bootloader 和 App 编译为**同一个固件镜像**，烧录后上电即运行。

### 首次烧录注意事项

- 首次烧录需**全片擦除**后再写入，保证 Bootloader 和 App 区域都正确初始化。
- 后续可通过串口 Bootloader 升级 App（不需要 ST-Link），协议帧格式见 `系统方案设计书.md` 第八章。

### newlib 警告

链接阶段出现 `_write`、`_read`、`_close` 等警告是 bare-metal 裸机程序的正常现象（`--specs=nosys.specs` 提供空桩实现），不影响固件功能。
